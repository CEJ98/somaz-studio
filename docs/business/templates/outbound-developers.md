# Outbound Sequence — Developers

## Perfil objetivo

- developer chico / mediano
- proyecto en etapa visible
- packaging debil o presentacion mejorable

## Email 1

Hola, [Nombre].

Estuve viendo [proyecto / empresa / desarrollo].

Vi una oportunidad clara: cuando un proyecto todavia necesita alinear inversores, permisos, clientes o stakeholders, un paquete visual flojo suele atrasar decisiones que ya deberian estar cerradas.

En Somaz trabajamos justamente esa capa: direccion arquitectonica, visualizacion para aprobaciones o presentaciones, e integracion con architect-of-record local cuando hace falta.

Si tiene sentido, puedo compartirte en una llamada corta como estructurariamos un package de clarity / investor / permit support para algo de este tipo.
