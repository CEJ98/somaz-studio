# Client Experience Playbook

## Objetivo

Diseñar una experiencia premium, clara y repetible desde el primer contacto hasta la postventa, sin volver pesada una operacion founder-led.

## Principios

- claridad antes que espectacularidad
- criterio arquitectonico en cada interaccion
- cercania sin perder trazabilidad
- tiempos visibles y promesas realistas
- una sola version vigente por etapa

## Canal oficial v1

- `WhatsApp` = cercania, coordinacion rapida, recordatorios
- `email` = trazabilidad, decisiones, propuestas, entregas, approvals

Reglas:

- ninguna aprobacion critica vive solo en WhatsApp
- si el feedback llega disperso, Somaz lo consolida y lo devuelve en formato unico
- si cambia una fecha o aparece un riesgo, se comunica antes de que el cliente lo persiga

## Dos tracks de experiencia

### Track A — developers / hospitality / partners

Prioridades:

- velocidad
- claridad entre stakeholders
- materiales circulables
- tono ejecutivo

Ajustes:

- updates mas breves y orientados a hitos
- foco en decisiones, riesgos y dependencias
- entregables escritos para circular internamente

### Track B — homeowners / clientes directos

Prioridades:

- contencion
- claridad del proceso
- traduccion de complejidad
- mas seguridad al decidir

Ajustes:

- mas contexto sobre que esperar
- feedback mas guiado
- menos jerga tecnica cuando no aporta

## Journey oficial

### 1. Primer contacto

Objetivo:

- que el lead se sienta recibido con criterio

SLA:

- `<= 24h habiles`
- high-fit o referral caliente: ideal `<= 6h habiles`

Senal premium:

- respuesta humana
- siguiente paso claro
- sin promesas vacias

### 2. Respuesta inicial

Debe confirmar:

- recepcion
- comprension basica del proyecto
- siguiente paso correcto
- timing esperado

Posibles salidas:

- `discovery_call`
- `paid_consult`
- `qualify_async`
- salida limpia si no hay fit

### 3. Discovery / consult

Discovery:

- `15-20 min`
- no es consultoria gratis

Paid consult:

- para proyectos borrosos, valiosos o multi-stakeholder

El cliente debe salir sintiendo:

- entendieron el proyecto
- saben ordenar el proceso
- no estan improvisando

### 4. Propuesta

Debe sentirse como documento de decision.

Incluye:

- alcance
- entregables
- exclusiones
- timeline
- rounds
- fee
- forma de pago
- compliance / jurisdiccion

### 5. Cierre + anticipo

No se inicia sin:

- contrato correcto
- anticipo
- partner path cuando aplica

Senal premium:

- el proyecto entra formalmente
- la transicion entre venta y ejecucion es limpia

### 6. Onboarding

Objetivo:

- bajar ansiedad
- aumentar control

Debe incluir:

- mensaje de bienvenida
- checklist
- canal principal
- kickoff
- lista de insumos
- hitos inmediatos

### 7. Delivery y revisiones

Reglas:

- un canal principal de feedback
- una sola consolidacion por ronda
- una sola version vigente
- fuera de alcance = `change_order`

Ritmo:

- update corto semanal si el proyecto sigue activo
- aviso adicional si cambia fecha, dependencia o riesgo

### 8. Entrega final / handoff

Diferenciar siempre:

- `final_deliverable`
- `client_facing_package`
- `partner_coordination_package`
- `non_submittable_base_package`

La entrega debe incluir:

- archivos ordenados
- naming consistente
- nota de entrega
- aclaracion de uso/compliance si aplica

### 9. Postventa

Timing:

- `3-7 dias`: follow-up corto
- `2-4 semanas`: feedback, referral y upsell natural si corresponde

Regla:

- respetar NDA y restricciones de portfolio antes de pedir referral o testimonial

## Tiempos de comunicacion

- respuesta comercial inicial: `<= 24h habiles`
- confirmacion de recepcion de feedback: mismo dia habil
- respuesta a dudas activas: `<= 1 dia habil`
- aviso de atraso o riesgo: anticipado

## Assets obligatorios

- `welcome email`
- `project roadmap / timeline summary`
- `kickoff confirmation`
- `feedback instructions`
- `revision summary`
- `delivery note`
- `post-project follow-up`
- `testimonial / referral ask`
- `client-facing status update`
- `delay / risk communication`

## Rituales internos

### Pre-kickoff review

- scope correcto
- canal principal
- stakeholders
- partner path
- hitos de primera semana

### Weekly client-facing update

- estado actual
- que se completo
- que sigue
- bloqueos o riesgos
- decision requerida si aplica

### Pre-delivery QA

- entregable correcto
- naming correcto
- disclaimer correcto
- proximo paso claro

### Closeout review

- aprobacion final
- saldo
- NDA / portfolio
- referral potencial
- upsell potencial
